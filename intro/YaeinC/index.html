목표
  • 연령대 누구나 사용하기 어렵지 않도록 쉬운 UI/UX를 갖는다
  • 페이지별 프린트 기능을 넣는다
  • 멤버 조회를 통하여 원하는 색인을 하는 열람기능을 제공한다
  • 출석 정보를 기록하여 통계화한다
기술
  • Infra : AWS ec2
  • Backend : Express(API server) + mongoDB
  • Front : React + Redux + scss(responsive) + bootstrap
    • 회고

    1. 풀스택 개발

    npm init부터 시작하여 front, back, infra를 다양하게 경험했습니다
    서버 코드는 어떻게 작성되는지, API는 어떤 패턴으로 설계되는지, 배포는 어떻게 하는지
    진귀한 경험을 해보는 시간이었습니다 웹개발에 자신감이 생기는 시간이었습니다

    2. 의존성 없는 개발

    컴포넌트와 템플릿을 개발하면서 의존성을 끊는 노력을 많이 했습니다
    관련하여 많이 배우기는 했지만 실제로 재사용을 고려할 수 밖에 없는 상황에 닥치니
    왜 필요한지 절실히 느끼며 개발하게 되었습니다
    개발할 때 이것을 항상 신경쓰며 개발하게 되었습니다
    함수형 프로그래밍에 관심을 갖게 되는 시간이 되기도 했습니다

    3. 라이브러리 사용 자주 하자

    sortable.js를 적용했는데 처음에 익숙하지가 않았습니다 라이브러리를
    많이 사용해보지 않아 어렵다는 느낌이 컸습니다
    라이브러리 의존적이면 안 된다는 것은 괜한 걱정이었습니다
    잘 사용하는 것도 하나의 능력인데 그것을 간과했었습니다
    앞으로는 유용한 라이브러리를 적극 도입해볼 예정입니다
